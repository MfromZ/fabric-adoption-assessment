# Change Management Research

Source: https://learn.microsoft.com/en-us/power-bi/guidance/fabric-adoption-roadmap-change-management

## Definition
Procedures to address the impact of change for people in the organization, safeguarding against disruption and productivity loss.

## Key Areas
- Communication of changes
- Training for new processes/tools
- Deployment pipelines (dev → test → prod)
- Version control
- Testing and validation
- Rollback procedures

## Maturity Levels
| Level | State |
|-------|-------|
| 100 | Changes happen without planning or communication, ad-hoc deployments |
| 200 | Some change processes emerging, limited communication about changes |
| 300 | Defined change management processes, deployment pipelines in use |
| 400 | Well-established practices, stakeholder communication, testing processes |
| 500 | Optimized with automation (CI/CD), proactive communication, minimal disruption |
